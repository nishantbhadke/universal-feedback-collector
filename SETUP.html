Universal Feedback Collector Setup Guide

Configure environment variables, connect Google Sheets, bootstrap nodemailer SMTP alert notification servers, and spin up active dev pipelines.

🚀 Developer Zero-Config Quickstart

The application is engineered with an abstract storage factory. Out of the box, with **zero configuration or external API keys**, it runs on a robust, persistent local JSON database fallback at .data/db.json. You can test submissions, upvotes, dashboard setups, and console emails instantly!

Start Development Server:

npm run dev
# Opens interface at: http://localhost:3000

⚙️ Environment Configurations (.env.local)

Create a file named .env.local in the root folder of the project to activate premium production synchronization (Google Sheets, NodeMailer SMTP notifications, Admin Passphrases): 

# ==============================================================================
# UPRC FEEDBACK COLLECTOR SYSTEM ENVIRONMENT CONFIGURATIONS
# ==============================================================================

# 1. ADMIN PASSPHRASE CONTROL (Default: admin123)
ADMIN_PASSPHRASE=admin123

# 2. PHASE 1: GOOGLE SHEETS ACTIVE ENGINE
# Input Service Account and Spreadsheet keys to disable Local Fallback
GOOGLE_SERVICE_ACCOUNT_EMAIL=your-service-account@your-project.iam.gserviceaccount.com
GOOGLE_PRIVATE_KEY="-----BEGIN PRIVATE KEY-----\nMIIEvgIBADANBgkqhkiG9w0BAQEFAASCBKgwggSkAgEAAoIBAQC...\n-----END PRIVATE KEY-----\n"
GOOGLE_SHEET_ID=your-google-spreadsheet-id-from-url

# 3. SMTP EMAIL NOTIFICATIONS ENGINE
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_USER=your-smtp-email@gmail.com
SMTP_PASS=your-smtp-app-password
SMTP_FROM="Universal Feedback Collector" <noreply@collector.com>
NOTIFICATION_RECIPIENT=your-alert-recipient@email.com

📊 Google Sheets Database Configuration

How to connect with Google Service Account:

  1. Create a Google Cloud Platform project and enable the **Google Sheets API**.
  2. Navigate to **Credentials**, create a **Service Account** key in JSON format, and save the email and private key.
  3. Create a blank Google Sheet in your Google Drive.
  4. Click **Share** in the sheet, paste the Service Account Email, and grant it **Editor** permissions.
  5. Copy the Spreadsheet ID from the URL (the string between /d/ and /edit) and add it to your .env.local.
  6. Self-Bootstrapping: The application automatically creates worksheets (tabs) like _Projects, _Categories, and dynamic tables per project on launch.

🚀 Vercel Production Hosting Deployment

The application is 100% optimized and structured to build cleanly into static HTML nodes and API Route Lambdas on Vercel:

  1. Push this independent repository to your separate GitHub account.
  2. Connect the repository to your **Vercel** dashboard.
  3. Inject all environmental variables from your .env.local under Vercel **Environment Variables** settings (especially replace double-escaped newlines in your private key safely).
  4. Click **Deploy**! Vercel handles compilation, caching, and edge dispatches automatically!
